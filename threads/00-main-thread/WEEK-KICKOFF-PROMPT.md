# Week Kickoff Prompt

Use this prompt at the start of any week to generate or refresh the week-specific execution file.

## Prompt

```text

Base it on:
- /Users/pawan/Vizmo/team-ai/threads/00-main-thread/WEEK-BY-WEEK.md
- /Users/pawan/Vizmo/team-ai/threads/04-resource-map/RESOURCE_AUDIT.md
- /Users/pawan/Vizmo/team-ai/threads/01-ai-engineering/README.md
- /Users/pawan/Vizmo/team-ai/threads/02-system-design/README.md
- /Users/pawan/Vizmo/team-ai/src/README.md
- /Users/pawan/Vizmo/team-ai/src/MONOREPO.md

Fill all 8 parts:
1. week goal
2. code slice
3. staff artifact
4. AI learning slice
5. system design drills
6. metrics or eval slice
7. tracking updates
8. end-of-week review

For this week:
- choose one code slice
- choose one staff artifact
- choose one AI learning slice
- choose one general system design drill
- choose one AI system design drill
- choose one metrics or eval slice
- choose only 2-3 resources for the week

The generated weekly doc should include:
1. Week Goal
2. Existing Baseline
3. Code Slice
4. Staff Artifact
5. AI Learning Slice
6. System Design Drills
7. Metrics Or Eval Slice
8. Day-By-Day Plan
9. Acceptance Criteria
10. Tracking Instructions

Keep the week scoped tightly. Prefer one end-to-end product slice over broad coverage.
```

## Rule

Do not start a week by gathering more random resources.
Start by locking:

- the code slice
- the artifact
- the learning slice
- the drills
- the resource pack

Then build against that scope for the rest of the week.
