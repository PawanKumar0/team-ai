# Start Each Week

This is the exact sequence to start any week.

If you only want one file that tells you what to do on Monday morning, use this one.

## Step 1: Open The Weekly Hub

Open these in order:

1. [WEEKLY-HUB.md](/Users/pawan/Vizmo/team-ai/threads/00-main-thread/WEEKLY-HUB.md)
2. [WEEKLY-RESOURCE-PACKS.md](/Users/pawan/Vizmo/team-ai/threads/04-resource-map/WEEKLY-RESOURCE-PACKS.md)
3. your current `week-N/README.md`
4. [threads/01-ai-engineering/WEEKS.md](/Users/pawan/Vizmo/team-ai/threads/01-ai-engineering/WEEKS.md)
5. [threads/02-system-design/WEEKS.md](/Users/pawan/Vizmo/team-ai/threads/02-system-design/WEEKS.md)
6. [threads/03-staff-engineering/WEEKS.md](/Users/pawan/Vizmo/team-ai/threads/03-staff-engineering/WEEKS.md)

Only open job-hunt and interview-bank weekly maps when their activation weeks begin.

## Step 2: Lock This Week

Before you start learning or coding, lock these 8 items in the week file:

1. week goal
2. code slice
3. staff artifact
4. AI learning slice
5. general system design drill
6. AI system design drill
7. metrics or eval slice
8. end-of-week review target

If one of these is missing, the week is not ready.

## Step 3: Lock The Resource Pack

Use the current week’s section in:

- [WEEKLY-RESOURCE-PACKS.md](/Users/pawan/Vizmo/team-ai/threads/04-resource-map/WEEKLY-RESOURCE-PACKS.md)

Pick:

- 2 to 4 primary resources
- 1 optional supplement at most

Do not keep adding random links during the week.

## Step 4: Start With The Artifact

Write or update the week’s artifact first:

- product brief
- architecture note
- ADR
- RFC
- risk register
- postmortem

This clarifies the build before code expands.

## Step 5: Build The Code Slice

Implement one end-to-end slice in `src/`.

Rule:

- one vertical slice
- one real user-visible outcome
- no framework wandering

## Step 6: Do The Learning Through The Build

Do not separate “learning time” and “building time” too much.
Learn the exact topics needed for the code slice and the two design drills.

## Step 7: Record The Week

Update:

- [TRACKER.md](/Users/pawan/Vizmo/team-ai/threads/00-main-thread/TRACKER.md)
- your current `week-N/README.md`
- weekly review in `journal/`

Daily logs are optional if you prefer a weekly update style, but the weekly review is mandatory.

## Step 8: End The Week Properly

At the end of the week, answer:

- what shipped?
- what was learned?
- what was measured?
- what is still weak?
- is the week actually complete?

Do not move to the next week because time passed.
Move only when the exit criteria are real.
